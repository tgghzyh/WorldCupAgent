## 一、产品定位（Product Positioning）

### 一句话价值主张

> **Not only predicts the World Cup champion — explains every prediction.**

### 展开描述

WC2026 是一个 AI Agent，它每天自动运行，生成世界杯每场比赛的预测结果。更重要的是，它能够向任何人（球迷、评委、甚至非技术背景的决策者）解释：为什么 AI 认为 A 队会赢？

这不是一个数据展示网站。
这是一个**会说话、会解释、会思考**的 AI 产品。

## 二、目标用户（Persona）

### 2.1 Persona 1 — 评委（Primary User）

```
姓名：     评委
场景：     打开演示链接 / 扫码进入网站
目标：     快速判断"这是一个真实工作的 AI Agent"
心理状态：  怀疑 → 好奇 → 信任

停留时间：  3 分钟
关键行为：
  1. 看首页：确认这是 AI 预测
  2. 点一场比赛：看 AI 怎么解释预测
  3. 回到首页：确认每天自动更新

成功标准：
  ✓ 5 秒内理解这是世界杯 AI 预测
  ✓ 30 秒内看到今天的预测结论
  ✓ 3 分钟内完成一次完整演示路径
  ✓ 离开时相信"这个 AI 是真实工作的"
```

### 2.2 Persona 2 — 世界杯球迷

```
姓名：     球迷
场景：     刷到链接 / 朋友分享
目标：     知道今天哪支队会赢
心理状态：  娱乐心态 → 参考心态

停留时间：  30 秒
关键行为：
  1. 扫一眼首页：看到 Featured Match
  2. 点击：看预测结论
  3. 离开或分享

成功标准：
  ✓ 10 秒内知道今天最重要比赛是什么
  ✓ 20 秒内知道 AI 的预测结论
  ✓ 愿意点击查看详情或分享
```

### 2.3 Persona 3 — 技术评审

```
姓名：     技术评审
场景：     检查项目实现质量
目标：     确认 Agent 架构合理性、代码质量
心理状态：  批判 → 审视 → 评估

停留时间：  5 分钟
关键行为：
  1. 读 About 页：了解 Agent 工作流
  2. 点开一场比赛：看 Factor Attribution
  3. 检查 Snapshot 版本：确认是动态数据

成功标准：
  ✓ 能说清楚"这个 Agent 是怎么预测的"
  ✓ 能看到 Factor 归因逻辑
  ✓ 能验证数据是动态生成的而非写死
```

## 四、首页 Hero 设计目标（5 个元素）

### 设计原则

**第一屏只允许出现 5 个元素。**

多余的元素 = 注意力分散 = 评委失去兴趣

### 允许的元素

```
① LOGO + 产品名
   "🤖 WC2026 AI Prediction Agent"

② 今日 AI 预测 Headline
   "Spain has a 94% chance of beating Qatar today."

③ Featured Match（最核心的比赛）
   - Argentina vs Mexico
   - 胜率条
   - [查看完整预测] CTA

④ 状态指示（Trust Layer）
   🟢 Live  |  72 Matches  |  Updated 2026-07-05 00:15 UTC

⑤ 导航入口（极简）
   [Groups]  [Tournament]  [About]
```

### 禁止的元素

```
❌ 第二场比赛卡片（除非是 Featured Match 的备选）
❌ ELO 排行榜（会分散注意力）
❌ 统计数据（属于技术评审页面的内容）
❌ 搜索框（这个产品不需要搜索）
❌ 语言切换（不需要）
❌ 任何"了解更多"链接
```

## 五、每个页面的核心任务

### 5.1 Home — 回答一个问题

> **"今天 AI 觉得谁能赢？"**

不是：
- "这里有多少数据"
- "这里有多少支球队"
- "你可以按组别筛选"

是：
- 今天最重要的一场比赛是什么？
- AI 的预测结论是什么？

### 5.2 Match — 回答一个问题

> **"AI 为什么这么预测？"**

不是：
- "这场比赛的历史战绩"（球迷会感兴趣，但评委更关心 AI）
- "赔率对比"
- "伤病信息"

是：
- AI 根据什么得出 79.8% 这个数字？
- 哪个因子影响最大？
- Agent 经历了哪几步思考？

### 5.3 Tournament — 回答一个问题

> **"整个赛程的 AI 预测是什么？"**

不是：
- "积分榜"（这只是信息）
- "历史数据"（技术评审才关心）

是：
- AI 认为谁会赢得每个小组？
- AI 认为哪支队会进入决赛？
- 淘汰赛的对阵预测是什么？

### 5.4 Compare — 回答一个问题

> **"AI 今天改变了什么预测？"**

不是：
- "历史对比"
- "统计分析"

是：
- 相比昨天，AI 对这场比赛的预测变了吗？
- 变化的原因是什么？（可以关联 Factor 变化）

### 5.5 About（Footer） — 回答一个问题

> **"这个 AI 是怎么工作的？"**

一句话：
> "每天，Agent 自动运行 6 步：计划 → 搜索 → 加载数据 → 推理 → 预测 → 保存。"

不展开技术细节，只给出一个清晰的"AI 在做什么"的感觉。

## 七、Trust Layer（信任建立层）

### 7.1 什么是 Trust Layer？

Trust Layer 是**散布在产品各处的"真实性信号"**，让用户（尤其是评委）相信：
- 这个 AI 是真实运行的
- 数据是动态生成的
- 预测不是写死的

### 7.2 Trust Layer 位置

```
位置 1: 首页右上角（固定）
┌─────────────────────────────────────┐
│ Snapshot  2026-07-05                │
│ v0.2.0  ·  Generated 00:15 UTC     │
│ 🟢 High Confidence                 │
└─────────────────────────────────────┘

位置 2: Match 页面 Meta Bar
"模型: ELO System v1  ·  训练样本: 448 场  ·  🟢 高可信"

位置 3: Snapshot Metadata Card
"Snapshot ID: snap_2026_07_05_0015
Expires: 2026-07-05 23:59 UTC
来源: worldcup_agent v0.2.0"

位置 4: Agent Timeline
"✅ Plan  ✅ Search  ✅ Load Data  ✅ Reason  ✅ Predict  ✅ Save"
```

### 7.3 Trust Layer 设计原则

| 原则 | 说明 |
|---|---|
| **始终可见** | 固定位置，不随滚动消失 |
| **具体数字** | 用精确时间戳，不用"刚刚更新" |
| **版本可追溯** | 每次预测都有 snapshot ID |
| **诚实降级** | 无数据时显示真实状态，不伪造 |

## 八、AI Confidence Center（AI 可信度中心）

### 8.1 概念

**不是简单地显示"HIGH / MEDIUM / LOW"，而是告诉用户"为什么是 HIGH"。**

### 8.2 设计形式

```
┌─────────────────────────────────────────────────────────────┐
│  Prediction Confidence                                     │
│                                                             │
│      87                                                    │
│      High                                                  │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  为什么是 87 分？                                    │  │
│  │                                                     │  │
│  │  ✓ ELO 数据新鲜度: 100%  (7天前更新)               │  │
│  │  ✓ 训练集规模: 448 场  (足够样本)                  │  │
│  │  ✓ 因子归因完整度: 100%  (2 个因子)                │  │
│  │  ⚠ 淘汰赛数据: 无  (小组赛阶段)                   │  │
│  └─────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 8.3 Confidence Score 计算逻辑（不对用户显示公式）

| 因子 | 权重 |
|---|---|
| ELO 数据新鲜度 | 30% |
| 训练集规模 | 20% |
| 因子归因完整度 | 20% |
| 淘汰赛数据存在性 | 15% |
| 预测一致性（与赔率对比） | 15% |
